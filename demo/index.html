제1과목 · 컴퓨터 일반

1. 자료의 표현 · 코드 체계 빈출▾

  • 자료 구성 단위(작은→큰): 비트 → 니블(4비트) → 바이트 → 워드 → 필드 → 레코드 → 파일 → 데이터베이스. 필드=파일 구성의 최소 단위.
  • 비트/바이트/워드: 바이트=문자 표현 최소 단위. 워드=CPU가 한 번에 처리하는 단위. 니블=4비트=16진수 1자리.
  • BCD: 6비트, 64문자, 영문 소문자 표현 불가.
  • EBCDIC: BCD 확장 8비트, 256문자, 대형 컴퓨터용.
  • ASCII 빈출: 7비트 128문자(데이터 통신·PC용). 확장 ASCII=8비트 256문자.
  • Unicode: 국제 표준, 세계 각국 언어를 2바이트(16비트)로 표현.
  • 실수형 데이터: 부호부+지수부+가수부. 정수형보다 표현 범위 크고 연산 속도 느림.
  • 10진 연산: 팩(Pack)·언팩(Unpack) 방식. 뺄셈은 보수와 덧셈으로 수행.
ASCII는 "각국 언어 표현" 불가 → 그건 유니코드. / Unicode를 4바이트라 하면 오답(2바이트). / 실수형을 "부호+가수부만"이라 하면 오답.

  • 연산장치 구성요소 빈출: 가산기(Adder), 보수기(Complementor, 뺄셈용), 누산기, 상태 레지스터, 데이터 레지스터.

3. 기억장치 빈출▾

  • 가상 메모리 빈출: 보조기억장치 일부를 주기억장치처럼 사용 → 주기억장치보다 큰 프로그램 실행.
  • 캐시 메모리 빈출: CPU와 주기억장치 사이, 두 장치 속도 차이를 줄여 처리 속도 향상(주로 SRAM).
  • 연관(Associative) 메모리: 주소가 아닌 내용(데이터)으로 접근. (보조기억을 주기억처럼 쓰는 건 가상 메모리)
  • RAM: 휘발성. DRAM=재충전(Refresh) 필요·주기억용, SRAM=캐시용·더 빠름. ROM=비휘발성·BIOS/펌웨어 저장.
  • SSD: 반도체(플래시) 기반, 기계적 장치 없음, 고속·저소음·충격에 강함. HDD보다 용량당 가격 비쌈.
  • 플래시 메모리: 비휘발성, 디지털카메라·MP3·USB 등 휴대기기용.
  • HDD 인터페이스: EIDE, SATA, SCSI.
  • 채널(Channel): CPU와 입출력장치 사이 속도 차이 해결(입출력 전담). DMA: CPU 개입 없이 메모리↔입출력 직접 전송.
DRAM 접근 속도(ns)는 작을수록 좋음. / "인터럽트가 속도 차이를 해결" → 오답(그건 채널). / SSD를 "충격에 약함·HDD보다 저렴" → 오답.

4. RAID · 시스템 유형 · 운영체제 빈출▾

  • RAID 빈출: 여러 HDD를 하나처럼. 데이터 복구 용이·안전성 높음(서버용). Level 숫자는 구성 방식 구분일 뿐(작을수록 신뢰성 높다 = 오답). 미러링=복제, 스트라이핑=분산 기록(속도↑).
  • 다중 처리(Multi-Processing) 빈출: 하나의 컴퓨터에 여러 CPU 설치, 주기억·주변장치 공유, 신뢰성·연산능력 향상.
  • 다중 프로그래밍: 한 CPU가 여러 프로그램을 번갈아 처리.
  • 시분할: 시간을 잘게 나눠 여러 사용자에게 할당. 실시간: 입력 즉시 처리(온라인). 일괄: 모아서 처리. 듀플렉스: 한 CPU 대기.
  • 임베디드 시스템: 특정 목적용 소형 컴퓨터(HW+SW 일체), 가전제품에 사용.
  • 인터럽트 빈출: 실행 중 예기치 않은 상황 발생 시 작업을 일시 중단·우선 처리 후 복귀.
  • 교착 상태(Deadlock): 둘 이상의 프로세스가 서로의 자원을 요구하며 무한 대기.
  • OS 성능 평가 기준 빈출: 처리 능력(Throughput)↑, 응답/반환 시간↓, 사용 가능도↑, 신뢰도↑.
다중 처리(여러 CPU) ↔ 다중 프로그래밍(한 CPU, 여러 프로그램) 구분이 핵심 함정.

5. 소프트웨어 · 프로그래밍 언어 빈출▾

  • 객체 지향 언어 빈출: 데이터+함수를 객체로 묶음. 상속·캡슐화·추상화·다형성, 정보 은폐·재사용 용이. 대표 C++, Java, C#. (BASIC·Pascal·C = 절차적 언어)
  • Java: OS 독립, 멀티스레드·분산처리. (인공지능·기호 처리는 LISP)
  • 번역 방식: 컴파일러(전체 번역→목적프로그램), 인터프리터(행 단위 번역). 원시→목적 변환은 처리 프로그램.
  • 시스템 SW=제어 프로그램(감시·작업관리·데이터관리)+처리 프로그램(언어번역·서비스·문제). (특정 업무용은 응용 SW)

  • BIOS 빈출: 기본 입출력, ROM 저장(펌웨어), 전원 켜면 가장 먼저 POST 수행, 칩 교환 없이 업그레이드 가능, CMOS로 설정.
  • 펌웨어(Firmware): 하드웨어 제어용, 주로 ROM에 반영구 저장, SW 업그레이드로 기능 향상. (운영체제는 아님)

6. 한글 Windows 10 빈출▾

  • 레지스트리 빈출: Windows 구성 정보 DB, 편집기 regedit.exe, 프로필·하드웨어·설치정보 포함, 수시로 참조(부팅 시에만 = 오답).
  • NTFS: 액세스 제어·제한 계정 지원, FAT32보다 성능·보안·안전성↑, 파일 크기는 볼륨 크기로만 제한. (모든 드라이브용 범용은 아님)
  • 휴지통 빈출: 복원=[복원] 메뉴/드래그, 드라이브별 크기(MB) 설정, 초과 시 오래된 파일부터 삭제. 위치 C:\$Recycle.Bin.
  • 휴지통에 안 가는 삭제: 네트워크 드라이브·USB·[Shift]+Delete·명령 프롬프트 삭제.
  • CMOS 설정 빈출: 날짜·시간, HDD 타입, 부팅 순서, 시스템 암호, 칩셋, PnP, Anti-Virus. BIOS 종류·Windows 로그인 암호는 설정 불가.
  • 글꼴: 위치 C:\Windows\Fonts, 확장자 .ttf/.otf/.ttc/.fon (.rtf·.inf 아님), ClearType로 가독성↑.
  • 연결 프로그램: 삭제해도 연결 정보만 삭제, 데이터 파일은 유지.
  • 파일 복사/이동: 같은 드라이브=드래그 이동/[Ctrl]+드래그 복사, 다른 드라이브=드래그 복사([Shift]+드래그 이동).
  • 여유 공간 부족 해결: 휴지통 비우기·디스크 정리·불필요 파일 삭제. (조각 모음은 속도 향상용, 용량과 무관)
  • 인쇄 작업 빈출: 대기 순서 조정·중지·재개 가능. 인쇄 중 문서는 다른 프린터로 전송 불가.
  • 바로 가기 키: Alt+Enter(속성) · Alt+F4(닫기) · Ctrl+Esc(시작) · Alt+PrintScreen(활성창 캡처) · Win+Tab(작업 보기).

7. 인터넷 · 네트워크 빈출▾

  • IPv6 빈출: 128비트, 16비트씩 8부분, 콜론(:) 구분. 유니캐스트·멀티캐스트·애니캐스트 3종(Broadcast 없음). 인증·기밀·무결성 지원, IPv4보다 빠름·호환.
  • TCP/IP: TCP=흐름 제어·오류 검사·패킷 분할/조립, IP=주소 지정·경로 결정.
  • 고정 IP 설정 항목: IP 주소·서브넷 마스크·게이트웨이·DNS 서버 주소. (DHCP 서버 주소는 불필요)

라우터서로 다른 네트워크 간 최적 경로 설정
브리지두 LAN 접속, 데이터 링크 계층에서 네트워크 확장

  • OSI 7계층 빈출: 물리(기계·전기·기능·절차) → 데이터링크(오류/흐름 제어) → 네트워크(경로 설정) → 전송 → 세션(대화 제어) → 표현(코드 변환·암호화·압축) → 응용(사용자 서비스).
  • DNS: 문자 도메인 ↔ IP 주소 변환, 계층적 관리. (최단 경로 설정은 라우터)
  • 전자우편: SMTP(발송), POP3/IMAP(수신), MIME(멀티미디어). 주소 ID@호스트. 7비트 ASCII 사용.
  • FTP: 파일 전송/삭제/이름변경. 그림=Binary, 텍스트=ASCII 모드. 서버 프로그램은 다운로드 후 실행.
  • 프로토콜 3요소: 구문(Syntax)·의미(Semantics)·순서(Timing). 기능=흐름 제어·동기화·오류 검출.
  • 기타 용어: LBS(위치 기반), 텔레매틱스(자동차+정보통신), IoT(모든 사물 연결), 테더링(폰을 모뎀처럼), 스트리밍(전송되는 대로 재생), 인트라넷(내부용).
텔레매틱스 ↔ LBS 혼동 주의. / IPv6를 "세미콜론 구분·64비트·Broadcast 포함" → 모두 오답. / 프로토콜에 "하드웨어 연결 문제 해결"은 기능 아님.

8. 정보 보안 빈출▾

  • 보안 위협 4유형: 가로막기(가용성↓)·가로채기(도청, 기밀성↓)·수정(무결성↓)·위조(인증↓).
  • 보안 요건 빈출: 기밀성·무결성·가용성·인증·부인 방지. (확장성은 아님)
  • 공격 유형: 피싱(가짜 사이트 유인), 스니핑(패킷 엿봄), 스푸핑(위장), DoS(대량 전송으로 마비), 혹스(악성코드인 척), 키로거.
  • 방화벽 빈출: 외부 침입 차단·역추적. 내부로부터의 불법 해킹은 못 막음.

바이러스 감염 증상: 파일 크기 커지고 속도 느려짐(작아지고 빨라진다 = 오답). / 저작권법은 프로그램 언어·규약·해법에는 적용 안 됨.
