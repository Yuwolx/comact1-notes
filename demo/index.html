제1과목 · 컴퓨터 일반

1. 자료의 표현 · 코드 체계 빈출▾

컴퓨터는 모든 자료를 0과 1로만 저장합니다. 그래서 "몇 비트를 묶어 한 글자로 볼 것인가"라는 약속이 필요한데, 그 약속이 바로 코드 체계입니다. 비트가 모여 문자가 되고 문자가 모여 파일이 되는 흐름부터 잡아 두면 나머지는 자연스럽게 따라옵니다.

  • 자료 구성 단위는 작은 것부터 큰 것 순서로 비트 → 니블(4비트) → 바이트 → 워드 → 필드 → 레코드 → 파일 → 데이터베이스입니다. 이 가운데 필드가 파일을 구성하는 최소 단위라는 점을 기억해 두세요.
  • 비트·바이트·워드를 구분해 봅시다. 바이트는 문자를 표현하는 최소 단위이고, 워드는 CPU가 한 번에 처리하는 단위입니다. 니블은 4비트짜리 묶음인데, 16진수 한 자리가 정확히 니블 하나에 대응합니다.
  • BCD 코드는 한 글자를 6비트로 표현합니다. 6비트로 만들 수 있는 조합이 64가지뿐이라 64문자가 한계고, 그래서 영문 소문자는 표현하지 못합니다.
  • EBCDIC 코드는 BCD를 8비트로 확장한 코드입니다. 비트가 늘어난 만큼 표현 가능한 문자도 256개로 늘었고, 주로 대형 컴퓨터에서 사용됩니다.
  • ASCII 코드 빈출는 7비트로 128문자를 표현하며, 데이터 통신과 PC에서 널리 쓰입니다. 여기에 1비트를 더한 확장 ASCII는 8비트로 256문자까지 표현할 수 있습니다.
  • 유니코드(Unicode)는 나라마다 제각각이던 문자 표현을 하나로 통일한 국제 표준입니다. 세계 각국의 언어를 모두 2바이트(16비트)로 표현합니다.
  • 실수형 데이터는 부호부·지수부·가수부의 세 부분으로 구성됩니다. 지수부 덕분에 정수형보다 표현할 수 있는 범위는 크지만, 그만큼 연산 속도는 느립니다.
  • 10진 연산에는 팩(Pack)과 언팩(Unpack) 방식이 있습니다. 뺄셈은 직접 빼는 대신 보수를 구해 덧셈으로 처리합니다.
ASCII로 "세계 각국의 언어를 표현할 수 있다"고 하면 오답입니다 — 그것은 유니코드의 특징입니다. 유니코드를 4바이트라고 해도 오답이고(정답은 2바이트), 실수형이 "부호부+가수부만"으로 구성된다는 설명도 오답입니다.

2. 중앙처리장치 · 레지스터 빈출▾

CPU가 계산을 하려면 명령과 데이터를 잠시 올려 둘 메모지가 필요합니다. 그 메모지 역할을 하는 CPU 안의 작은 임시 저장 공간이 레지스터입니다. 시험에서는 각 레지스터가 "무엇을 기억하는지"를 서로 바꿔 놓고 물어봅니다.

레지스터역할
명령 레지스터(IR)현재 실행 중인 명령을 기억합니다
프로그램 카운터(PC)다음에 실행할 명령의 주소를 기억합니다 (명령의 내용이 아니라 주소입니다)
누산기(ACC)연산 결과를 일시적으로 저장합니다
데이터 레지스터연산에 사용할 데이터를 기억합니다
상태 레지스터연산 중 발생하는 상태값을 기억합니다
  • 연산장치의 구성요소 빈출로는 덧셈을 맡는 가산기(Adder), 뺄셈을 위해 보수를 만들어 주는 보수기(Complementor), 그리고 누산기·상태 레지스터·데이터 레지스터가 있습니다.
  • 마이크로프로세서는 제어장치·연산장치·레지스터를 칩 하나에 모두 담은 것입니다. 성능은 클럭 주파수와 버스 폭으로 평가하는데, 주기억장치는 이 칩에 포함되지 않는다는 점에 주의하세요.

3. 기억장치 빈출▾

기억장치는 빠를수록 비싸고, 쌀수록 느립니다. 그래서 컴퓨터는 빠른 장치와 느린 장치 사이사이에 캐시나 가상 메모리 같은 "중간 다리"를 놓아 속도 차이를 메웁니다. 이 큰 그림을 잡고 각 장치의 위치와 역할을 살펴봅시다.

  • 가상 메모리 빈출는 보조기억장치의 일부를 주기억장치처럼 사용하는 기법입니다. 덕분에 주기억장치 용량보다 큰 프로그램도 실행할 수 있습니다.
  • 캐시 메모리 빈출는 CPU와 주기억장치 사이에 자리 잡고 두 장치의 속도 차이를 줄여 전체 처리 속도를 높입니다. 주로 SRAM으로 만듭니다.
  • 연관(Associative) 메모리는 주소가 아니라 내용(데이터)으로 접근하는 메모리입니다. "보조기억장치를 주기억장치처럼 쓴다"는 설명이 붙어 있다면 연관 메모리가 아니라 가상 메모리를 말하는 것입니다.
  • RAM은 전원이 꺼지면 내용이 사라지는 휘발성 메모리입니다. DRAM은 재충전(Refresh)이 필요하고 주기억장치에 쓰이며, SRAM은 더 빨라서 캐시에 쓰입니다. 반대로 ROM은 비휘발성이라 BIOS·펌웨어를 저장하는 데 사용됩니다.
  • SSD는 반도체(플래시) 기반이라 기계적으로 움직이는 부품이 없습니다. 그래서 빠르고 소음이 없으며 충격에도 강하지만, 같은 용량이면 HDD보다 가격이 비쌉니다.
  • 플래시 메모리는 전원이 없어도 내용이 유지되는 비휘발성 메모리로, 디지털카메라·MP3·USB 같은 휴대기기에 쓰입니다.
  • HDD 인터페이스에는 EIDE, SATA, SCSI가 있습니다.
  • 채널(Channel)은 입출력을 전담하면서 CPU와 입출력장치 사이의 속도 차이를 해결해 줍니다. DMA는 CPU의 개입 없이 메모리와 입출력장치가 직접 데이터를 주고받는 방식입니다.
DRAM의 접근 속도(ns)는 숫자가 작을수록 빠른 것입니다. "인터럽트가 속도 차이를 해결한다"는 오답이고(그 역할은 채널), SSD를 "충격에 약하다·HDD보다 저렴하다"고 해도 오답입니다.

4. RAID · 시스템 유형 · 운영체제 빈출▾

컴퓨터는 부품을 하나만 써야 한다는 법이 없습니다. 디스크를 여러 개 묶으면 RAID, CPU를 여러 개 달면 다중 처리가 됩니다. "무엇을 여러 개 쓰는가"를 기준으로 시스템 유형을 구분하면 헷갈릴 일이 줄어듭니다.

  • RAID 빈출는 여러 개의 HDD를 하나의 디스크처럼 묶어 쓰는 기술입니다. 데이터 복구가 쉽고 안전성이 높아 서버에 많이 사용됩니다. Level 뒤의 숫자는 구성 방식을 구분하는 번호일 뿐이므로 "숫자가 작을수록 신뢰성이 높다"고 하면 오답입니다. 미러링은 같은 데이터를 복제해 두는 방식이고, 스트라이핑은 데이터를 분산해 기록함으로써 속도를 높이는 방식입니다.
  • 다중 처리(Multi-Processing) 빈출는 한 대의 컴퓨터에 CPU를 여러 개 설치하는 방식입니다. CPU들이 주기억장치와 주변장치를 공유하며, 신뢰성과 연산 능력이 향상됩니다.
  • 다중 프로그래밍은 CPU가 하나인데, 그 하나가 여러 프로그램을 번갈아 가며 처리하는 방식입니다.
  • 시분할 시스템은 CPU의 시간을 잘게 나눠 여러 사용자에게 할당합니다. 실시간 시스템은 입력이 들어오는 즉시 처리하고(온라인), 일괄 처리는 자료를 모아 두었다가 한꺼번에 처리합니다. 듀플렉스는 CPU 한쪽을 대기시켜 두는 방식입니다.
  • 임베디드 시스템은 특정 목적만을 위해 만든 소형 컴퓨터로, 하드웨어와 소프트웨어가 하나로 묶여 있습니다. 가전제품에 널리 쓰입니다.
  • 인터럽트 빈출는 프로그램 실행 중 예기치 않은 상황이 생겼을 때, 하던 작업을 잠시 멈추고 그 상황을 먼저 처리한 뒤 원래 작업으로 되돌아오는 것입니다.
  • 교착 상태(Deadlock)는 둘 이상의 프로세스가 서로가 가진 자원을 요구하면서 무한정 기다리는 상태입니다.
  • 운영체제의 성능 평가 기준 빈출은 네 가지입니다. 처리 능력(Throughput)은 높을수록, 응답 시간과 반환 시간은 짧을수록, 사용 가능도와 신뢰도는 높을수록 좋은 운영체제입니다.
다중 처리는 "CPU 여러 개", 다중 프로그래밍은 "CPU 하나로 여러 프로그램"입니다. 이 둘을 바꿔 놓는 것이 단골 함정입니다.

5. 소프트웨어 · 프로그래밍 언어 빈출▾

소프트웨어는 컴퓨터 자체를 움직이는 시스템 소프트웨어와, 우리가 실제 일을 하는 데 쓰는 응용 소프트웨어로 나뉩니다. 여기에 사람의 말을 컴퓨터의 말로 바꿔 주는 프로그래밍 언어와 번역기까지 한 번에 묶어 정리하는 단원입니다.

  • 객체 지향 언어 빈출는 데이터와 함수를 하나의 객체로 묶어서 다루는 언어입니다. 상속·캡슐화·추상화·다형성이라는 특징이 있고, 그 덕분에 정보 은폐와 재사용이 쉽습니다. 대표 언어는 C++, Java, C#이며, BASIC·Pascal·C는 객체 지향이 아니라 절차적 언어입니다.
  • Java는 운영체제에 상관없이 실행되며, 멀티스레드와 분산처리를 지원합니다. 인공지능·기호 처리용 언어를 묻는다면 Java가 아니라 LISP가 답입니다.
  • 번역 방식은 두 갈래입니다. 컴파일러는 프로그램 전체를 한 번에 번역해 목적 프로그램을 만들고, 인터프리터는 한 행씩 번역하면서 실행합니다. 원시 프로그램을 목적 프로그램으로 바꾸는 일은 처리 프로그램이 담당합니다.
  • 시스템 소프트웨어는 제어 프로그램과 처리 프로그램으로 구성됩니다. 제어 프로그램에는 감시·작업 관리·데이터 관리가 있고, 처리 프로그램에는 언어 번역·서비스·문제 프로그램이 있습니다. 특정 업무를 처리하려고 만든 프로그램은 시스템 소프트웨어가 아니라 응용 소프트웨어입니다.

  • BIOS 빈출는 기본 입출력을 담당하는 프로그램으로, ROM에 저장된 펌웨어입니다. 전원을 켜면 가장 먼저 실행되어 POST를 수행합니다. 칩을 교환하지 않고도 업그레이드할 수 있으며, 설정은 CMOS에서 합니다.
  • 펌웨어(Firmware)는 하드웨어를 제어하기 위한 소프트웨어로, 주로 ROM에 반영구적으로 저장됩니다. 소프트웨어 업그레이드만으로 기능을 향상시킬 수 있지만, 펌웨어를 운영체제라고 부르면 틀립니다.

6. 한글 Windows 10 빈출▾

운영체제인 Windows가 설정과 파일을 어떻게 관리하는지 다루는 단원입니다. 평소 컴퓨터를 쓰며 겪은 경험 그대로라 어렵지 않지만, 시험은 "휴지통을 거치지 않는 삭제"처럼 예외 상황을 집중적으로 묻습니다.

  • 레지스트리 빈출는 Windows의 각종 구성 정보를 모아 둔 데이터베이스입니다. 사용자 프로필과 하드웨어, 설치 정보가 여기에 담기며, 편집기 regedit.exe로 내용을 고칠 수 있습니다. 시스템이 수시로 참조한다는 점이 핵심으로, "부팅 시에만 참조한다"고 하면 오답입니다.
  • NTFS는 Windows의 파일 시스템입니다. 액세스 제어와 제한된 계정을 지원하고, FAT32보다 성능·보안·안전성이 뛰어납니다. 파일 크기는 볼륨 크기로만 제한됩니다. 다만 모든 드라이브에 쓰는 범용 방식은 아니라는 점을 기억하세요.
  • 휴지통 빈출은 삭제한 파일이 잠시 머무는 공간입니다. [복원] 메뉴나 드래그로 되살릴 수 있고, 크기는 드라이브별로 MB 단위로 설정합니다. 용량이 가득 차면 오래된 파일부터 지워지며, 실제 위치는 C:\$Recycle.Bin입니다.
  • 휴지통에 안 가는 삭제: 네트워크 드라이브나 USB에서 지운 파일, [Shift]+Delete로 지운 파일, 명령 프롬프트에서 삭제한 파일은 휴지통을 거치지 않고 바로 사라집니다.
  • CMOS 설정 빈출은 부팅 전에 하드웨어의 기본값을 정하는 화면입니다. 날짜·시간, HDD 타입, 부팅 순서, 시스템 암호, 칩셋, PnP, Anti-Virus를 여기서 설정할 수 있습니다. BIOS 종류와 Windows 로그인 암호는 설정할 수 없습니다.
  • 글꼴 파일은 C:\Windows\Fonts 폴더에 저장됩니다. 확장자는 .ttf/.otf/.ttc/.fon이고(.rtf·.inf는 글꼴이 아닙니다), ClearType 기능을 켜면 화면 글자의 가독성이 좋아집니다.
  • 연결 프로그램을 삭제하면 지워지는 것은 연결 정보뿐입니다. 데이터 파일 자체는 그대로 남습니다.
  • 파일 복사/이동: 같은 드라이브 안에서는 그냥 드래그하면 이동이 되고, [Ctrl]을 누른 채 드래그하면 복사가 됩니다. 다른 드라이브로 옮길 때는 반대로, 드래그가 복사이고 [Shift]+드래그가 이동입니다.
  • 여유 공간 부족 해결: 디스크가 꽉 찼다면 휴지통 비우기, 디스크 정리, 불필요한 파일 삭제로 공간을 확보합니다. 조각 모음은 속도 향상용일 뿐이어서 용량 확보와는 관계가 없습니다.
  • 인쇄 작업 빈출: 인쇄 대기열에서는 대기 순서 조정, 중지, 재개가 모두 가능합니다. 단, 이미 인쇄 중인 문서를 다른 프린터로 보낼 수는 없습니다.
  • 바로 가기 키: Alt+Enter는 속성 창, Alt+F4는 창 닫기, Ctrl+Esc는 시작 메뉴를 엽니다. Alt+PrintScreen은 활성 창을 캡처하고, Win+Tab은 작업 보기를 띄웁니다.

7. 인터넷 · 네트워크 빈출▾

인터넷은 전 세계 컴퓨터가 약속된 규칙(프로토콜)에 따라 데이터를 주고받는 거대한 네트워크입니다. 이 단원에서는 컴퓨터의 주소 체계(IP), 통신 규칙, 그리고 데이터를 중간에서 전달해 주는 장비들을 정리합니다.

  • IPv6 빈출는 주소를 128비트로 늘린 체계입니다. 128비트를 16비트씩 8부분으로 나누고, 각 부분을 콜론(:)으로 구분해 적습니다. 주소 유형은 유니캐스트·멀티캐스트·애니캐스트 3종뿐이며 Broadcast는 없습니다. 인증·기밀성·무결성을 지원하고, IPv4보다 빠르며 서로 호환됩니다.
  • TCP/IP는 인터넷의 기본 프로토콜입니다. TCP는 흐름 제어·오류 검사·패킷의 분할과 조립을 맡고, IP는 주소 지정과 경로 결정을 담당합니다.
  • 고정 IP 설정 항목: IP를 직접 지정하려면 IP 주소, 서브넷 마스크, 게이트웨이, DNS 서버 주소를 입력합니다. DHCP 서버 주소는 입력할 필요가 없습니다.
장비기능
라우터서로 다른 네트워크 사이에서 최적 경로를 설정합니다
브리지두 LAN을 접속하고, 데이터 링크 계층에서 네트워크를 확장합니다
게이트웨이현재 네트워크에서 다른 네트워크로 연결해 줍니다
리피터약해진 신호를 증폭하여 재생합니다
프록시 서버방화벽 기능과 캐시 기능을 함께 제공합니다
  • OSI 7계층 빈출은 통신 과정을 7단계로 나눈 표준 모형입니다. 물리(기계·전기·기능·절차) → 데이터링크(오류/흐름 제어) → 네트워크(경로 설정) → 전송 → 세션(대화 제어) → 표현(코드 변환·암호화·압축) → 응용(사용자 서비스) 순서로 이어집니다.
  • DNS는 문자로 된 도메인 이름과 IP 주소를 서로 변환해 주는 시스템이며, 계층적으로 관리됩니다. 최단 경로 설정은 DNS가 아니라 라우터의 일입니다.
  • 전자우편은 보낼 때 SMTP, 받을 때 POP3나 IMAP을 사용합니다. 멀티미디어 첨부는 MIME이 처리하고, 주소는 ID@호스트 형식이며 7비트 ASCII를 사용합니다.
  • FTP는 파일을 전송·삭제하거나 이름을 바꿀 수 있는 서비스입니다. 그림 파일은 Binary 모드로, 텍스트 파일은 ASCII 모드로 전송합니다. 서버에 있는 프로그램은 그 자리에서 실행하는 것이 아니라, 다운로드한 뒤에 실행해야 합니다.
  • 프로토콜 3요소: 통신 약속은 구문(Syntax)·의미(Semantics)·순서(Timing)의 세 요소로 이루어집니다. 기능으로는 흐름 제어, 동기화, 오류 검출이 있습니다.
  • 기타 용어: LBS는 위치 기반 서비스이고, 텔레매틱스는 자동차에 정보통신을 결합한 기술입니다. IoT는 모든 사물을 연결하는 개념이며, 테더링은 스마트폰을 모뎀처럼 쓰는 기능입니다. 스트리밍은 파일이 전송되는 대로 바로 재생하는 방식이고, 인트라넷은 조직 내부용 네트워크를 말합니다.
텔레매틱스와 LBS를 혼동하지 않도록 주의하세요. IPv6를 "세미콜론 구분·64비트·Broadcast 포함"으로 설명하면 모두 오답입니다. "하드웨어 연결 문제 해결"은 프로토콜의 기능이 아닙니다.

8. 정보 보안 빈출▾

보안은 결국 "누가, 어떤 방식으로 데이터를 노리는가"와 "그것을 어떻게 막는가"의 이야기입니다. 위협 유형별로 어떤 보안 요건이 깨지는지 짝지어 두면 문제 대부분이 풀립니다.

  • 보안 위협 4유형: 가로막기는 데이터의 흐름 자체를 막아 가용성을 떨어뜨립니다. 가로채기는 몰래 도청하는 것으로 기밀성을 해칩니다. 수정은 무결성을 깨뜨리고, 위조는 인증을 위협합니다.
  • 보안 요건 빈출: 안전한 시스템이 갖춰야 할 요건은 기밀성·무결성·가용성·인증·부인 방지입니다. 확장성은 보안 요건이 아닙니다.
  • 공격 유형: 피싱은 가짜 사이트로 유인해 정보를 빼내는 수법입니다. 스니핑은 오가는 패킷을 몰래 엿보고, 스푸핑은 다른 대상인 것처럼 위장합니다. DoS는 대량의 데이터를 한꺼번에 보내 시스템을 마비시키며, 혹스는 악성코드인 척하는 가짜입니다. 이 밖에 키로거도 알아 두어야 합니다.
  • 방화벽 빈출은 외부에서 들어오는 침입을 차단하고 역추적할 수 있습니다. 하지만 내부로부터의 불법 해킹은 막지 못합니다.
암호화특징
비밀키(대칭, DES)암호화와 복호화에 같은 키를 씁니다. 알고리즘이 단순해 속도가 빠른 대신, 키 개수가 많아집니다
공개키(비대칭, RSA)공개키로 암호화하고 비밀키로 복호화합니다. 키 관리가 쉽지만 알고리즘이 복잡해 속도가 느립니다
바이러스에 감염되면 파일 크기가 커지고 속도는 느려집니다("작아지고 빨라진다"는 오답). 저작권법은 프로그램 언어·규약·해법에는 적용되지 않습니다.

9. 멀티미디어 · 그래픽 빈출▾

글·소리·그림·영상을 한데 묶어 다루는 것이 멀티미디어입니다. 이 단원의 핵심은 그림을 표현하는 두 방식(비트맵과 벡터)의 차이, 그리고 소리를 디지털로 바꾸는 원리입니다.

  • 멀티미디어 특징 빈출: 여러 매체를 하나로 다루는 통합성과, 데이터를 디지털로 처리하는 디지털화가 기본입니다. 여기에 사용자의 선택에 따라 다양한 방향으로 진행되는 비선형성, 그리고 쌍방향으로 주고받는 상호 작용성이 더해집니다.
구분비트맵(래스터)벡터
표현점(픽셀)으로 표현점·직선·곡선(베지어)으로 표현

용량큼작음
형식BMP·TIF·GIF·JPEG·PNGAI·WMF
  • 파일 형식: JPEG는 사진 압축의 표준으로, 손실 압축과 무손실 압축을 모두 지원합니다. GIF는 256색까지 표현하며 애니메이션이 가능하고, PNG는 트루컬러와 투명을 지원합니다. BMP는 압축을 하지 않아 파일 크기가 큽니다.
  • 그래픽 기법: 렌더링은 명암과 색을 입혀 사실감을 살리는 기법입니다. 안티앨리어싱은 계단 현상을 없애 주고, 모델링은 물체의 형상을 정의합니다. 모핑은 두 이미지를 자연스럽게 변환하는 기법이며, 디더링은 제한된 색을 조합하는 기법입니다.
  • 사운드 빈출: 샘플링은 아날로그 소리를 디지털로 바꾸는 과정입니다. 샘플링률(Hz)과 비트가 높을수록 원음에 가깝고 음질이 좋아집니다.
  • mp3 크기 결정: mp3 파일의 크기는 표본 추출률(Hz), 샘플 크기(Bit), 시간, 재생 방식(Mono/Stereo)으로 정해집니다. 프레임 너비(Pixel)는 소리 파일과 무관합니다.
"직선·곡선으로 이미지를 표현한다"를 비트맵 설명에 넣으면 오답입니다(벡터의 특징). BMP를 "크기가 작다"고 해도 오답입니다. 샘플링 주파수는 "낮을수록 좋다"가 아니라 높을수록 좋습니다.
